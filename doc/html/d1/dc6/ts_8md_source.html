1 # Tree sequences
2 
3 fwdpp 0.7.0 introduced support for generating "succinct tree sequences" during simulations. See \cite Kelleher2016-cb
4 for background on the data structures and \cite Kelleher2018-fu on their application to forward simulation.
5 
6 Some readers will probably find the [msprime documentation](https://msprime.readthedocs.io/en/stable/) helpful here, as
7 many of the big-picture concepts are the same. The relevant section of that documentation is titled "Tree sequence interchange".
8 
9 ## Example programs
10 
11 I encourage you to read the [tutorials](https://tskit-dev.github.io/tutorials/) that we have written for the tskit project. The tutorials on forward-time simulation give an overview of the logic of a simulation. While the overall idea of "generate nodes and edges, simplify every now and then, repeat" is straightforward, there are some gotchas that tripped up the authors of \cite Kelleher2018-fu more than a few times.
12 
13 1. wfts.cc implements a Wright-Fisher simulation with selection, ancestral sample recording, and ancestral sample
14  metadata tracking. Command line switches also enable extensive testing of the data structures.
15 2. spatialts.cc implements a simplistic method of simulating discrete generations on a continuous landscape. The
16  methods used in this example are **not** what one would do in research-quality code. Rather, the example is
17  illustrative, and shows how the geographic locations of preserved samples may be tracked.

19 ## Namespace
20 
21 Tree sequence support is found in namespace fwdpp::ts. Like the rest of fwdpp, it is header-only.

23 ## Data types
24 
25 1. Nodes and edges are defined by fwdpp::ts::node and fwdpp::ts::edge, respectively.
26 2. Mutations are tracked on trees via fwdpp::ts::mutation_record.
27 
28 Node, edge, and mutation tables are stored as vectors of the above three types.
29 
30 fwdpp::ts::table_collection holds node, edge, and mutation tables.
31 
32 fwdpp::ts::table_simplifier is responsible for applying the simplification algorithm of \cite Kelleher2018-fu to a
33 fwdpp::ts::table_collection.

35 ## Relationship to msprime/tskit
36 
37 I'll refer to *tskit* here, which means the part of msprime dealing with tree sequences.
38 
39 1. Node times are measured forwards in time. See fwdpp::ts::node.
40 2. The mutation table is simply a vector of records tracking nodes where mutations occur on the tree and their indexes in the population. See
41  fwdpp::ts::mutation_record. Thus, the concept of a "sites table" and a "mutation table" are merged in fwdpp.
42 3. The default storage format is a custom format. Once a standalone tskit C library is available, we will likely
43  provide direct access to storage in that tree file format as well. The current binary format is required for things
44  like MPI, Python pickling, etc., though, so it'll stick around.

46 ## Iterating over trees
47 
48 fwdpp::ts::tree_visitor iterates over the non-recombining segments ("marginal trees") in the pedigree. This type
49 defines a call operator that handles the iteration, updating an internal variable of type fwdpp::ts::marginal_tree.
50 
51 ![A simplified, fully-coalesced marginal tree](images/tree.png)
52 
53 Consider the tree shown above. It has 7 nodes. The parent list of the tree is represented as a linear vector
54 (fwdpp::ts::marginal_tree::parents), and would contain the following values:
55 
56 ```
57 4 4 5 5 6 6 fwdpp::ts::TS_NULL_NODE
58 ```
59 
60 The final value, which is the parent of the node labelled **6**, is fwdpp::ts::TS_NULL_NODE, which signifies that there
61 is no parent.
62 
63 Iteration over parents is trivial:
64 
65 ```cpp
66 void visit_parents(const fwdpp::ts::TS_NODE_INT n, const fwdpp::ts::marginal_tree & m)
67 {
68  auto p = n;
69  while(p != fwdpp::ts::TS_NULL_NODE)
70  {
71  p = m.parents[p];
72  }
73 }
74 ```
75 
76 Of course, a real-world application would have to provide bounds-checking, and would probably actually *do* something
77 with `p` at each iteration.
