15 /*! @defgroup samplingPops Functions related to taking samples from simulated
16  populations
17 
18  This collection of functions allows a user to draw a sample of size \f$n \ll
19  2N\f$ from
20  a simulated population.
21 
22  The library provides several overloads of the functions fwdpp::ms_sample and
23  fwdpp::ms_sample_separate.
24 
25  The following features are in common to all versions of these functions:
26 
27  1. They only supp]ort bi-allelic mutation positions.
28  2. All functions return some number of objects of type std::vector<
29  std::pair<double, std::string> >.
30  Each element in the vector corresponds to a mutation. The double corresponds
31  to the mutation position,
32  and the string corresponds to the character states in each of the \f$n\f$
33  samples. The strings contain the character
34  '0' when a sample has the ancestral state or '1' for the derived (mutant)
35  state. This type may be used
36  to populate a polymorphism table from the
37  [libsequence](http://molpopgen.github.io/libsequence/) library:
38  \code
39  auto x = fwdpp::ms_sample( appropriate arguments );
40  Sequence::SimData xx(x.begin(),x.end());
41  \endcode
42  When a SimData object is written to a stream, its output format is the same
43  as that used by Dick Hudson's coalescent simulation
44  program [ms](http://home.uchicago.edu/~rhudson1/source/mksamples.html)
45  3. All versions of fwdpp::ms_sample return vectors where the mutations
46  affecting fitness and those not affecting fitness are intermingled.
47  4. All versions of fwdpp::ms_sample_separate return pairs of vectors
48  separating the mutations not affecting fitness from those that do.
49  The first member of each pair is a vector of "neutral" mutations, and the
50  second member is the vector of "selected" mutations.
51  5. The functions are all implemented using sampling with replacement from the
52  population. Thus, setting \f$n = N\f$ (or \f$2N\f$) will NOT
53  return all the variable sites in the entire population!!!
54 
55  A vector may contain data looking like the following:
56  \verbatim
57  0.0760 "10000"
58  0.6805 "00010"
59  \endverbatim
60 
61  The two rows are the two different positions (0.076 and 0.6805). The first
62  haplotype is "10", the second is "00", and the fourth is "01".
63 
64  If you converted the data to a Sequence::SimData object and printed it to
65  screen or a file, the output would be:
66 
67  \verbatim
68  //
69  segsites: 2
70  positions: 0.0760 0.6805
71  10
72  00
73  00
74  01
75  00
76  \endverbatim
77 */
