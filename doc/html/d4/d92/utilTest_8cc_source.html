125 // The following unit tests are checks for GitHub issue #130
126 // This issue involves the situation where an extinct mutation
127 // has the same position as an extant mutation. Through fwdpp
128 // 0.6.0, this cause the extant mutation's position to
129 // be removed from the lookup table, which is a book-keeping bug.
130 // However, I ran a ton of sims and never found a difference due
131 // to the bug vs the fixed version (fwdpp 0.6.1). The reason
132 // is that, for the bug to have an effect, a series of rare events
133 // must happen multiple times.
