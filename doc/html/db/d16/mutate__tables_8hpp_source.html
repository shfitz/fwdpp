21  /// \brief Apply a mutation scheme to add neutral mutations to a fwdpp::ts::table_collection.
22  ///
23  /// \param r fwdpp::GSLrng_t
24  /// \param make_mutation A mutation function. See below.
25  /// \param samples A list of sample nodes corresponding to "currently-alive" nodes
26  /// \param mu Mutation rate (per gamete, per generation)
27  ///
28  /// \version 0.7.0 Added to library
29  ///
30  /// This function uses the edge_table to apply mutations to all parent/child connections.
31  /// The mutations should be neutral, although that requirement is not enforced. (It simply
32  /// makes little sense to apply selected mutations to the entire edge table post-hoc.)
33  ///
34  /// The result of this function is to populate @tables.mutation_table with neutral variants.
35  ///
36  /// The parameter \a make_mutation is a function that must conform to
37  /// std::function<std::size_t(double, double, fwdpp::uint_t)>. The three arguments
38  /// are interpreted as "left", "right", and "time". The function's return value
39  /// corresponds to the index of the new mutation in the simulation's mutation container.
40  /// The new mutation must have a position on the half-open interval [left, right). The
41  /// "time" parameger represents the generation when the mutation arose and will be
42  /// uniformly assigned between the parental birth time and that of the child.
43  /// This mutation's origin time will be on the half-open interval (parental birth time,
44  /// child birth time].
45  ///
46  /// The preceding paragraph implies that \a make_mutation is repsonsible for required operations
47  /// related to mutation recycling, etc.. The lambda "neutral_variant_maker" in the example
48  /// program wfts.cc will probably be of help here.
49  ///
50  /// It is common that, at the end of a forward-time simulation, that not all trees are
51  /// completely coalesced. This happens because the distribution on the time to MRCA
52  /// has very high variance and, due to recombination, some fraction of trees will not be
53  /// coalesced after, say, 10N generations of evolving a single Wright-Fisher deme. Further,
54  /// the simlification algorithm will "push" the most ancient nodes on these "marginal forests"
55  /// forwards in time to the most recent ancestral node of each tree in the forest. Thus, naively
56  /// mutating the edge table will place too few mutations on these parts of the genome. This
57  /// function corrects for the presence of marginal forests, as described below.
58  ///
59  /// The \a samples list is used to identify "marginal forests", meaning marginal trees
60  /// that are not completely coalesced. (The finding is done via a call to
61  /// fwdpp::ts::mark_multiple_roots.) The MRCA nodes on trees in marginal forests have additional
62  /// mutations on them, representing evolution from time point zero (the beginning of the simulation)
63  /// to the MRCA node's time.
64  ///
65  /// Note that two alternatives exist that will render the treatment of marginal forests unnecessary:
66  /// 1. Simulate for longer. For example, CDF of TMRCA under Wright-Fisher is quite close to 1 at ~20N generations.
67  /// Thus, simulating longer means that fewer and fewer marginal trees will be forests.
68  /// 2. Start the simulation with an existing, completely-coalesced, tree sequence. As far as fwdpp is concerned,
69  /// this is left as an "exercise for the reader" at the moment.
70  {
71  unsigned nmuts = 0;
72  auto mr = mark_multiple_roots(tables, samples);
73  for (auto &i : mr)
74  {
75  auto dt = tables.node_table[i.first].time;
76  for (auto j : i.second)
77  {
78  double mean = dt * (j.second - j.first) * mu;
79  auto nm = gsl_ran_poisson(r.get(), mean);
80  nmuts += nm;
81  for (unsigned m = 0; m < nm; ++m)
82  {
83  unsigned g = static_cast<unsigned>(
84  gsl_ran_flat(r.get(), 1, dt + 1));
85  auto k
86  = make_mutation(j.first, j.second, g);
87  tables.mutation_table.emplace_back(
88  mutation_record{ i.first, k });
89  }
90  }
91  }
92  for (auto &e : tables.edge_table)
93  {
94  auto ct = tables.node_table[e.child].time;
95  auto pt = tables.node_table[e.parent].time;
96  auto dt = ct - pt;
97  double mean = dt * (e.right - e.left) * mu;
98  auto nm = gsl_ran_poisson(r.get(), mean);
99  for (unsigned m = 0; m < nm; ++m)
100  {
101  unsigned g = static_cast<unsigned>(
102  gsl_ran_flat(r.get(), pt + 1, ct + 1));
103  auto k = make_mutation(e.left, e.right, g);
104  tables.mutation_table.emplace_back(
105  mutation_record{ e.child, k });
106  }
107  nmuts += nm;
108  }
109  return nmuts;
110  }
